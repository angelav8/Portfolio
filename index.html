Pacman

A fun game developed in C#

I created this game as part of a group for a project for a university class. I love playing games, so this was a really fun project to develop. This project was created by a team of three, myself and two friends. We found this project quite challenging, as it was our first time implementing advanced features such as collision detection, custom event handlers to handle how the lives and scores were changed. Overall, our game worked almost perfect.

  • Objectives:
  • Using advanced Object Oriented Programming such as inheritance to build a console application.
  • Using Event Handling to change the score, lives left, etc.
  • Movement of sprites and their animation
  • Advanced features such as being able to change the colour scheme of the game, adding extra levels, power ups when Pacman eats certain pills

  • Comments
  • The game was a successthe interface looked very well, the opening menu and winning/losing pages.
  • The collision between pacman, and the ghosts was perfect, with the score and lives updating when a pill is picked up/live lost when ghost touches pacman, and a sound is played each time they collide, too.
  • All characters were animated when moving

Shape View Project

A project created using C# in Visual Studio.

This project is my first project using C#. This was created by myself and a friend. It is created so that the user will be able to draw different shapes onto a windows form-based application. The user can specify whether they would like to draw an Eclipse, a Rectangle or a Right-Angled Triangle, once the shape is picked the user can change the width and height using separate sliders. The application will then work out the circumference and the area of the shape. The user can also select whether or not they would like the figures shown in 2, 3 or 4 decimal points.

Being new at the time to C#, I found this project quite challenging. We split the work into different sections and worked together continuously until the project was completed. Using a paired-programming approach, we were able to work together very well. Overall, the project was a great success, although we had a few problems coming towards finishing up the program.

Objectives:

  • To make a Windows form-based application demonstrating your programming ability in dealing with events in C#
  • Using advanced Object-Oriented Programming features such as inheritance
  • Practice programming ability demonstrated in the use of form controls/components

Comments:

  • The layout is easy to read and the shapes draw on the canvas flawlessly
  • More inheritance could have been used. If I were to do this project again, I would break it down into different classes which are related. This would organise my code much better instead of having it on one long class, and one or two smaller methods implemented in another class.
  • The layout was not dynamic Although I made sure my shape would stay drawn when the window was resized, I should have put it into layout panel like a TableLayoutPanel. The shape and the control options were also cut off when the window was resized. If a layout panel was used this would not allow the application to be resized, therefore no controls would be cut off short.

Tile Game!

Final Year Project

A Raspberry Pi Garden Monitoring System

The project is a garden monitoring application where a user can check different statistics on their garden using an android application, connected to a raspberry pi system.

The idea was influenced from my dad, who has a passion for gardening. He owns an allotment situated 5 miles from our house. To reduce the amount of time he spends driving in and out from the allotment, I decided I would develop a system which allowed him and other users to check statistics on their garden such as whether their plants are watered, the temperature and the humidity of their garden.

The project consisted of a fully automated Raspberry Pi which had a soil moisture sensor and a temperature sensor attached. When he sensors detect input, the data is stored on a Google Firebase Realtime Database. When the Soil Moisture is detected as Dry, the user will receive a push notification telling them that their plants need watered.

The application also connects to a weather API which will display the weather on a screen for the user when requested. On another screen, the application connects to the Google Maps API where the user can view the nearest garden centres to them.

Within a different screen, the user can view a database of plants and what season they grow in. This is so the user can have an idea of which plants they can grow in their garden.
